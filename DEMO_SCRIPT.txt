Hello, my name is Rajath Goudar and this is my coursework submission for CS4S763 Mobile Application Development. My application is called SushiHub. It is a full stack sushi menu management app connected to a Node.js, Express and MongoDB API.

First, I will show authentication. I log in using the demo admin account. The backend checks the password and returns a JWT token. The frontend stores this token and uses it for protected API requests.

Now I will show the dashboard. It shows the number of menu items and confirms the API is active.

Next, I will show the read operation. The sushi menu page loads all records from the backend API and displays them as responsive cards.

Now I will create a new item. I enter a name, price, category, image URL and description. The app validates the fields and sends a POST request to the API. The new record appears in the menu and is stored in MongoDB.

Next, I will update the item. The edit form loads the existing data, I change the price or description, and submit it. This sends a PUT request to the backend.

Now I will delete the item. The app asks for confirmation and then sends a DELETE request. The item is removed from the database and the list refreshes.

Finally, I will show validation. If I try to submit an empty name or invalid price, the application prevents submission and shows an error message.

Overall, SushiHub demonstrates API communication, authentication, protected routes, CRUD operations, validation, error handling and responsive user interface design.
